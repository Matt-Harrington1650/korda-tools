# Workflows Overview

This page complements the in-app Workflow Map UI.

## Add a custom tool

1. Create tool metadata and instructions.
2. Attach one or more files.
3. Save version.
4. Export zip.
5. Share and import on another machine.

See [Tools Library](help://tools-library).

## Run a workflow

1. Select workflow.
2. Validate step tool bindings.
3. Start run.
4. Inspect node outputs and status.
5. Retry/fix failed steps.

## Update or version a tool

1. Open tool detail.
2. Add version with changelog and updated instructions.
3. Export the new version package.

## Troubleshoot issues

- Missing credentials
- Invalid file package
- Path or permissions issues
- Adapter/config mismatches

See [Troubleshooting](help://troubleshooting).
