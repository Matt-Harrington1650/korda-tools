# Repository Labels (v1)

| Label Name | Description | When To Use | Color (Hex) |
|---|---|---|---|
| `triage` | Requires initial review and routing. | New issue has not been classified. | `#BFD4F2` |
| `bug` | Verified defect in current behavior. | Reproducible bug reports. | `#D73A4A` |
| `enhancement` | Backward-compatible improvement. | Feature improvements without breaking behavior. | `#A2EEEF` |
| `feature` | Net-new capability. | Approved new capability work. | `#1D76DB` |
| `documentation` | Docs-only change. | Content or process documentation updates. | `#0075CA` |
| `security` | Security-sensitive change or issue. | Vulnerability, policy hardening, secret handling. | `#B60205` |
| `governance` | Policy/process/repo controls update. | Constitution, checklists, ownership, standards. | `#5319E7` |
| `audit` | Auditability/provenance/immutability work. | Audit chain, provenance, evidentiary controls. | `#FBCA04` |
| `compliance` | Legal/regulatory alignment work. | Regulatory mappings, control evidence changes. | `#0E8A16` |
| `ai-policy` | External AI usage policy impact. | Provider restrictions, override workflows, guardrails. | `#C5DEF5` |
| `breaking-change` | Incompatible behavior or API shift. | PR introduces MAJOR SemVer impact. | `#000000` |
| `release` | Release preparation or cut. | Changelog, version bump, tagging, release notes. | `#F9D0C4` |
| `priority:p0` | Critical priority. | Production-blocking or release-blocking work. | `#B60205` |
| `priority:p1` | High priority. | Important near-term work for current milestone. | `#D93F0B` |
| `priority:p2` | Medium priority. | Planned but not urgent work. | `#FBCA04` |
| `priority:p3` | Low priority. | Backlog or opportunistic improvements. | `#CCCCCC` |
